# ツギイチ AIアシスト仕様（Step分解）

## 目的
Goal（やりたいこと）を Step（次の一手）に分解する際、テンプレよりも状況に合った“初手”を提案し、予定化までの摩擦を減らす。

## 重要原則（審査/安全）
- AIはユーザーの明示操作（ボタンタップ）でのみ実行する（自動実行しない）
- 初回実行前に「外部AIに送信すること」「送信するデータ種別」「提供元」を明示し、明示的同意を得る（同意なし送信禁止）
- 送信するテキストをプレビュー表示し、任意でマスク（メール/電話/住所っぽい表現）できる
- AIが失敗してもテンプレ分解にフォールバックし、クラッシュしない

## UI/導線
### Goal詳細
- ボタン：『AIでステップ案を作る』
- 押下 → 送信内容プレビュー → 同意 → 生成 → Step一覧へ反映（編集可能）

### Settings
- AIアシスト：ON/OFF
- AIプロバイダ表示（例：OpenAI via 自前Proxy）
- “送信前に毎回確認”トグル（デフォルトON）
- “個人情報っぽい文字を自動マスク”トグル（デフォルトON）

## 送信データ（最小）
- goalTitle（必須）
- goalNote（任意）
- category（任意）
- constraints（任意：今週やりたい/予算/場所などユーザーが入力した場合のみ）

※位置情報や連絡先などは使わない（許諾と説明が重くなるため）

## 出力フォーマット（AIはJSONのみ返す）
{
  "steps": [
    {
      "title": "…",
      "type": "調べる|予約する|用意する|行く|作る|連絡する",
      "durationMin": 15|30|60|120,
      "dueSuggestion": "today|this_week|none",
      "notes": "任意"
    }
  ]
}

## 生成ポリシー
- stepsは3〜8件
- 最初の1件は「今日15分でできる初手」になるようにする
- 不確実な内容は“調べる”へ寄せる
- 断定しない（安全側の表現）

## 実装（iOS）
- SwiftUI + SwiftData
- AI呼び出しは URLSession
- 秘密鍵をアプリに同梱しない（クライアントにAPIキー埋め込み禁止）
- 推奨：自前の軽量Proxy（Cloudflare Workers等）にPOSTして結果を受け取る
